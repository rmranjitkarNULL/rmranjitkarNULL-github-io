Autonomous Trash-Collecting Romi

As part of RBE 2002: Unified Robotics II – Sensing and Perception, my team designed and programmed an autonomous Romi mobile robot capable of navigating a grid field, detecting AprilTags, collecting trash, and dumping it up a ramp. The system combined sensor fusion, finite state machines, timed control loops, and MQTT communication between multiple microcontrollers to achieve fully autonomous behavior. This project demonstrated complete integration of perception, control, and inter-device coordination within a constrained embedded environment.

Sensor Fusion and Perception

The Romi integrated a diverse sensor suite to enable robust autonomy through sensor fusion. Motor encoders provided wheel odometry for precise velocity and distance control, while a reflectance array handled line detection and intersection processing for grid navigation. A load cell paired with an AD620 instrumentation amplifier enabled accurate weight measurement, converting ADC readings to grams within ±1 g. The OpenMV camera communicated via I²C to detect AprilTags, providing positional data to guide pickup and dumping sequences. An onboard IMU measured acceleration and angular velocity, combined through a complementary filter to yield stable pitch and heading estimates. This continuous multi-sensor blending allowed the robot to interpret terrain changes, recognize orientation, and react to its environment in real time.

State Machines and Timed Control

The robot’s behavior was structured around a modular finite state machine (FSM) that governed navigation, perception, and task execution. Each state—such as LINE_FOLLOW, TURN, PICKUP, and DUMP—was time-synchronized using microcontroller timers to maintain consistent update rates for sensing and actuation loops. These timed intervals ensured deterministic control, enabling smooth PID-based line following and synchronized arm movement. By coupling the FSM with precise timing logic, the Romi could transition cleanly between tasks while preventing conflicts between parallel sensor and motion threads. This architecture also simplified debugging and scalability for additional mission steps.

MQTT Communication and Multi-Robot Coordination

A key system feature was the implementation of MQTT communication for message exchange between multiple ESP32 boards. A central “broker” node dispatched task updates and position data, while each Romi acted as a client subscribing to relevant topics. This lightweight publish-subscribe framework enabled distributed control, allowing robots to share progress states such as “trash collected” or “returning to start.” Through MQTT, the system maintained asynchronous coordination without blocking core control loops—demonstrating how embedded networking can extend autonomy beyond a single platform.

Testing, Results, and Lessons Learned

During the final showcase, the Romi successfully executed the full autonomous pipeline—navigating to the correct coordinates, recognizing AprilTags, collecting trash, climbing the ramp, and returning to the starting cell. While occasional line-sensor false positives and ramp slip occurred, performance was highly repeatable after PID and threshold refinements. The final system demonstrated the effectiveness of combining sensor fusion with FSM-based logic, timer-driven control, and MQTT communication in a unified architecture. The project solidified my understanding of embedded real-time design, multi-sensor calibration, and the coordination required for scalable robotic autonomy.
